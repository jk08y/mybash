# Contributing to mybash

First off, thank you for considering contributing to `mybash`. It's people like you that make mybash such a great tool.

### 1. Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check the [Issues](../../issues) page to see if someone else has already created a ticket. If not, go ahead and make one!

### 2. Fork & create a branch

If this is something you think you can fix, then fork mybash and create a branch with a descriptive name.

### 3. Get the test suite running

Make sure you run scripts like `shellcheck` against your scripts before pushing to ensure they meet our professional coding standards.

### 4. Implement your fix or feature

At this point, you're ready to make your changes. Feel free to ask for help; everyone is a beginner at first.

### 5. Make a Pull Request

At this point, you should switch back to your master branch and make sure it's up to date with mybash's master branch.
Then, create a pull request with a clear list of what you've done (read more about pull requests). When you write a PR, please make sure you include a clear explanation of what the change does.
